# Contributing to Klin

Thank you for contributing to Klin! Please follow these rules:

1. **Review Standards**: Read all guidelines under `standards/` before authoring new packages.
2. **Review Playbooks**: Follow the corresponding step-by-step tutorial under `playbooks/` (e.g. `playbooks/create-component.md`).
3. **Commit Messages**: Format commit messages according to `standards/naming.md` guidelines.
4. **Testing**: Write unit tests inside your component package folders.
