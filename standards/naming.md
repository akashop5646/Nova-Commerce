# Naming Conventions Standard

## Folders & Packages
- Use kebab-case for workspace directory layouts (e.g. `builder-core`, `puck-adapter`).
- Use lowercase for component folder packages (e.g. `packages/ui/button/`).

## React Components
- Use PascalCase for visual elements (`Button.tsx`, `Hero.tsx`).
- Use camelCase for schema definitions (`schema.ts`), configs (`config.ts`), and renderers (`render.ts`).

## Commit Guidelines
- Format commits following conventional-changelog specifications: `[type](scope): message` (e.g., `feat(ui/button): add disabled state`).
